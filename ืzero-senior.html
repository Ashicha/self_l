Set Up Your Environment
Start Here
🧑‍🏫

Beginnergit

Month 1–2 · Learn to think like a programmer
Programming Fundamentals
Foundation
🧑‍🏫

Pick one language and stick with it. I recommend Python — it reads like English and teaches concepts without fighting syntax. Don't try to learn multiple languages at once. You're not learning syntax, you're learning how to decompose problems. That skill transfers to every language.

✓
Variables, data types (int, string, bool, list, dict) — write 20 examples for each type
💡 Don't just read — TYPE the examples yourself. Your hands need to learn too.
Beginnerpython
✓
Control flow: if/else, for loops, while loops — build a number-guessing game
💡 Project: guess a random number 1–100, give "too high"/"too low" hints, count the tries.
Beginnerpython
✓
Functions: parameters, return values, scope — write 10 small, well-named reusable functions
💡 A function should do ONE thing. Name it clearly: get_user_age(), not process_stuff().
Beginnerpython
✓
Lists and dictionaries: slicing, methods, looping — build a contact book app in the terminal
💡 Project: add, remove, search contacts stored in a dict. Run from terminal, no UI needed.
Beginnerpython
✓
Error handling: try/except — learn to read stack traces. Errors are clues, not insults.
💡 Intentionally cause 5 different types of errors and read every line of the traceback carefully.
Beginnerpython
✓
File I/O: read and write text files and JSON — make your contact book persist between runs
💡 Save contacts to contacts.json on exit, load them on startup. Now your app has memory.
Moderatepython
✓
Complete 30 exercises on Exercism.io (Easy only) — solve each one 2 different ways
💡 Struggle is the point. It's literally building new neural pathways. The discomfort is the lesson.
Beginnerpractice
✓
Build Project #1: To-do CLI app — add, complete, delete, list tasks, saved to JSON. Push to GitHub.
💡 Your first real project on GitHub. You'll look back at this with pride. Write a README.
Moderateproject

Month 2–4 · Build things you can show people
Web Development Basics
Hands-On
🧑‍🏫

The web is the largest platform in human history. You need to understand HTML, CSS, and JavaScript — not to be a designer, but because every system you'll build touches the web. Also: building something you can open in a browser and show your family feels amazing and keeps you going when things get hard.

✓
HTML fundamentals: semantic tags, forms, links, images — build a personal bio page
💡 HTML is the skeleton. Use semantic tags: <nav>, <main>, <article> — not just <div> for everything.
Beginnerhtml
✓
CSS: box model, flexbox, grid, colours, fonts, responsive media queries
💡 Box model + flexbox unlocks 80% of all layouts. Master these two before anything else.
Moderatecss
✓
JavaScript basics: DOM manipulation, events, fetch API, async/await
💡 JS makes pages interactive. Start by making a button change text, then fetch data from an API.
Moderatejavascript
✓
Call a free public API (weather, jokes, Pokémon) and display live data on a webpage
💡 Try api.open-meteo.com (weather). This is how every modern app works — fetching live data.
Moderatejavascript
✓
Git branching: create a feature branch, make changes, merge back to main, resolve a conflict
💡 Branches = parallel universes for your code. Work on features without ever touching main.
Moderategit
✓
Deploy your first live website on GitHub Pages or Vercel (free)
💡 Your software is on the real internet. Share the URL. This moment matters more than you think.
Beginnerdeployment
✓
Build Project #2: Weather dashboard — search any city, show forecast, deployed live with a URL
💡 HTML + CSS + JS + public API + GitHub Pages. This is your first real portfolio piece.
Moderateproject

Month 4–7 · Where real apps are built
Backend & Databases
Core Skills
🧑‍🏫

Frontends are the face. Backends are the brain. This is where you learn to build APIs — the glue between apps and databases. By the end of this phase, you'll understand how every app you've ever used actually works under the hood. This phase will be hard. That's correct. Push through it.

✓
Understand HTTP deeply: GET, POST, PUT, DELETE, status codes (200, 404, 500, 401, 403)
💡 Open DevTools → Network tab on any website. Watch every single request your browser makes.
Moderatebackend
✓
Build a REST API with Express.js or FastAPI — full CRUD for a "notes" resource
💡 CRUD = Create, Read, Update, Delete. Build: POST /notes, GET /notes/:id, PUT, DELETE.
Moderatebackend
✓
SQL fundamentals: SELECT, INSERT, UPDATE, DELETE, WHERE, JOIN — start with SQLite (zero setup)
💡 Every database uses the same core SQL. Master it once, use it everywhere.
Moderatedatabases
✓
Database design: primary keys, foreign keys, relationships — design a schema for an e-commerce store
💡 Draw on paper: users, products, orders, order_items. What connects to what?
Moderatedatabases
✓
Connect your API to a real PostgreSQL database — no more data stored in arrays in memory
💡 Use Prisma (JS) or SQLAlchemy (Python). Save and retrieve real persistent data.
Moderatebackend
✓
Authentication: user sign-up, login, JWT tokens, bcrypt password hashing
💡 RULE #1: NEVER store plain-text passwords. Always bcrypt. No exceptions. Ever.
Challengingsecurity
✓
Test your API manually with Postman or Insomnia — test every route, including error cases
💡 Postman is your API workshop. Break things on purpose. What happens with bad inputs?
Beginnertesting
✓
Write your first automated unit tests — cover at least 5 functions with Jest or Pytest
💡 A test is: "if I give X, I expect Y." Start with pure functions — no database, no network.
Moderatetesting
✓
Build Project #3: Full-stack blog — React frontend + API backend + PostgreSQL + auth + deployed
💡 Users register, log in, create/edit/delete posts. This is your flagship portfolio piece.
Challengingproject

Month 7–10 · Think bigger than one file
CS Fundamentals & Clean Code
Level Up
🧑‍🏫

By now you can build things. Now we make you dangerous. CS fundamentals separate engineers who hit walls from those who break through them. You don't need a CS degree — but you need to understand how computers think. This phase also teaches you to write code other humans can actually read.

✓
Big-O Notation: understand O(1), O(n), O(n²), O(log n) — analyse your own existing code
💡 Ask: "If my input doubles, what happens to the runtime?" This is how you spot slow code before it hurts.
Moderatealgorithms
✓
Data structures: arrays, linked lists, stacks, queues, hash maps — implement each from scratch
💡 Implementing from scratch teaches you what your language hides. You'll understand libraries far better.
Moderatedata structures
✓
Trees and graphs: binary search trees, BFS, DFS — find a real-world use case for each
💡 File systems are trees. Social networks are graphs. You use these every day without knowing it.
Challengingdata structures
✓
Solve 40 LeetCode Easy problems — no looking at solutions for the first 30 minutes of each
💡 That discomfort is your brain building new circuits. Embrace it. Log every solution with notes.
Moderatepractice
✓
OOP: classes, inheritance, encapsulation, polymorphism — refactor your contact book using classes
💡 Take your Phase 1 project and rewrite it with proper OOP. Feel how structure changes everything.
Moderatedesign
✓
Clean Code: meaningful names, small functions, zero magic numbers — refactor a messy codebase
💡 Read the first 5 chapters of Clean Code by Robert Martin. They will permanently change how you code.
Moderatecode quality
✓
SOLID principles: read and find 1 real example of each in an open-source GitHub project
💡 Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
Challengingdesign
✓
Design patterns: implement Factory, Observer, and Strategy — use each in a small real program
💡 Patterns are reusable solutions to common problems. Seniors recognise them instantly.
Challengingdesign patterns
✓
Get your project code reviewed — fix every single piece of feedback, understand why
💡 Don't argue with feedback. Fix it first, understand it second, then decide if you agree. Humility scales.
Moderatecraft

Month 10–13 · Own the entire lifecycle
DevOps, Cloud & Systems Thinking
Senior Territory
🧑‍🏫

Senior engineers don't just write code — they own the entire lifecycle: build, test, deploy, monitor, fix at 3am. This phase will make you valuable in any company. Engineers who understand systems end-to-end are 10× more impactful than those who can only code locally.

✓
Docker: containerise your full-stack project — write a Dockerfile and docker-compose.yml
💡 "Works on my machine" ends with Docker. Containers run identically everywhere.
Moderatedocker
✓
CI/CD: set up GitHub Actions to auto-run tests and auto-deploy on every push to main
💡 Every push: tests run → build happens → deploy fires. Automated. Every time. No exceptions.
Moderatedevops
✓
Cloud basics (AWS free tier): deploy backend to EC2, store files in S3, understand IAM roles
💡 Start small. One EC2 instance, one S3 bucket. Feel how cloud infrastructure works.
Challengingcloud
✓
Intro to system design: design Pastebin from scratch — draw the full architecture
💡 Think: where is data stored, how does traffic reach the server, what if a million people use it?
Challengingsystem design
✓
Caching: add Redis to your project — use it for sessions, rate limiting, or frequently-read data
💡 Redis is lightning-fast in-memory storage. Cache expensive database queries to save milliseconds.
Challengingsystem design
✓
Security: rate limiting, input validation, SQL injection prevention, env vars for secrets
💡 Never trust user input. Never hardcode secrets. Parameterise all SQL. Never build your own crypto.
Challengingsecurity
✓
Monitoring: add structured logging and set up error alerts via Sentry (free tier)
💡 If your app breaks in production at 3am, you want to know before your users do.
Moderatemonitoring
✓
Build Project #4: Production-grade API — Docker + CI/CD + cloud deploy + monitoring + full test suite
💡 This runs in the cloud, tests itself on every push, and alerts you when it breaks. That's real engineering.

Month 13–18 · You know things. Now prove it.
Portfolio, Interviews & Landing the Job
Final Stretch
🧑‍🏫

The final phase is about translation — converting everything you know into opportunities. Interviewing is a skill separate from engineering, and you must practise it specifically. Your portfolio isn't a gallery; it's a proof of your judgement. Show problems you solved and decisions you made — not just a list of technologies.

✓
Solve 30 LeetCode Medium problems under time pressure (45 min each, no hints allowed)
💡 After each: write your approach, your mistakes, and the optimal solution in your own words.
Challenginginterviews
✓
System design practice: design YouTube, Uber, and WhatsApp — use the framework every time
💡 Framework: clarify scope → estimate scale → high-level design → deep dives → identify bottlenecks.
Challenginginterviews
✓
Behavioural prep: write 15 STAR stories covering failures, conflicts, ownership, and impact
💡 STAR = Situation, Task, Action, Result. Seniors are hired for judgment — your stories prove it.
Moderateinterviews
✓
Open source contribution: find a "good first issue", fix a bug or improve docs, get a PR merged
💡 Your first merged OSS PR is a career milestone. It shows you can work in a real codebase.
Moderateopen source
✓
Write and publish 5 technical blog posts about things you actually learned and struggled with
💡 "If you can't explain it simply, you don't understand it well enough." — Feynman. Teaching deepens learning.
Beginnerwriting
✓
GitHub portfolio: 3 pinned projects with READMEs that explain WHY you built it, not just what
💡 Bad: "A todo app built with React." Good: "I built this to learn state management and discovered..."
Beginnerportfolio
✓
Do 5 live mock interviews on Pramp.com — record yourself watching back is uncomfortable and invaluable
💡 Watching yourself code under pressure reveals habits you never knew you had.
Challenginginterviews
✓
Apply to 20 roles — treat every rejection as data, not a verdict. Ask for feedback when possible.
💡 Senior engineers fail interviews too. Every no is closer to a yes. The number is the strategy.
Moderatecareer
