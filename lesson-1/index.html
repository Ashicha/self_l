рҹ“Ў Part A: How Client & Server Communicate (HTTP)

The browser and server talk using HTTP (HyperText Transfer Protocol). It's like a conversation with strict rules:

STEP 1: Client sends a REQUEST в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ GET /api/flights?origin=BKK&dest=NRT HTTP/1.1 Host: localhost:3000 вҶ’ "Hey server, I want flights from BKK to NRT" STEP 2: Server processes the request в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ - Read origin=BKK, dest=NRT - Search the database - Find 4 matches STEP 3: Server sends a RESPONSE в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ HTTP/1.1 200 OK Content-Type: application/json { "count": 4, "flights": [ { "flightNumber": "TG601", "price": 12500 }, ... ] } вҶ’ "Here are 4 flights, as JSON data"

HTTP Status Codes вҖ” The Server's Vocabulary

The server responds with a number that tells the client what happened:

Code Meaning When Used Flowchart Example
200 OK вҖ” Success! Everything worked Flights found, booking confirmed
201 Created New resource created Seat held, booking created
400 Bad Request Client sent invalid data "Inputs valid?" вҶ’ No
404 Not Found Resource doesn't exist Flight ID doesn't exist
429 Too Many Requests Rate limited Your flowchart: "Rate limited?" вҶ’ {"error":"429"}
500 Internal Server Error Server crashed Something broke on our side
рҹ“– Reading status codes:
вҖў 2xx = Success (everything is fine) вң…
вҖў 4xx = Client error (YOU sent something wrong) вқҢ
вҖў 5xx = Server error (WE broke something) рҹ’Ҙ

рҹ“– Part B: Reading server.js вҖ” Line by Line

Let's read the server code the same way we learned in Module 1. Open module-2/server.js in VS Code and follow along.

1 Import & Setup

const express = require("express"); // Load the Express library const app = express(); // Create a new server app app.use(express.json()); // Enable reading JSON from requests
рҹ“– Reading these lines:
require("express") = "Go get the Express library from node_modules"
express() = "Create a new server application"
app.use() = "Add this feature/plugin to my server"

SA perspective: This is like your "System Initialization" section in a design doc.

2 Define a Route (API Endpoint)

// Pattern: app.METHOD(PATH, HANDLER) app.get("/api/flights", (req, res) => { // req = the incoming request (what client sent) // res = the response object (what we send back) const { origin, dest } = req.query; // Read ?origin=BKK&dest=NRT // Filter flights array (same as Module 1!) const results = flights.filter( flight => flight.origin === origin && flight.dest === dest ); res.json({ count: results.length, flights: results }); // Send JSON back });
рҹ’Ў Core Concept вҖ” Route = Flowchart Arrow:

Every arrow in your flowchart that says GET /api/... or POST /api/... becomes ONE route in your server code:

app.get("/api/flights", ...) вҶ’ handles the search
app.post("/api/seats/hold", ...) вҶ’ holds a seat (Module 3)
app.post("/api/promos/validate", ...) вҶ’ checks promo code (Module 7)
app.post("/api/bookings/confirm", ...) вҶ’ confirms booking (Module 7)

3 New Syntax: Arrow Functions & Destructuring

// ARROW FUNCTION: shorter way to write a function // These two are THE SAME: // Old way (Module 1): function search(req, res) { ... } // New way (arrow function): const search = (req, res) => { ... } // Even shorter (used directly in app.get): app.get("/api/flights", (req, res) => { ... })
// DESTRUCTURING: shortcut to extract object properties // These two are THE SAME: // Old way: const origin = req.query.origin; const dest = req.query.dest; // New way (destructuring): const { origin, dest } = req.query; // It "unpacks" the object into separate variables. // Think of it as: "Take origin and dest OUT OF req.query"
// .filter() вҖ” Array method that works like a loop + if // These two are THE SAME: // Module 1 way: const results = []; for (const flight of flights) { if (flight.origin === origin && flight.dest === dest) { results.push(flight); } } // Module 2 way (shorter, same result): const results = flights.filter( flight => flight.origin === origin && flight.dest === dest );
рҹ“– Reading .filter():
"Go through each flight. Keep only the ones where the condition is true."
It's the same loop+if pattern from Module 1, just written in one line.

4 Start the Server

A port = a "door number" on your computer.
Port 3000 means the server answers at http://localhost:3000
Like a restaurant вҖ” the building is your computer, port 3000 is the entrance door.
